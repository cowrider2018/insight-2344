# 2344 籌碼面深化 — 待辦計畫（改天執行）

## 背景與現況（執行前先讀）
- 系統現為**八面**（技術/籌碼/消息/基本/美光/費半/日內 1 分 K/主力分點），確定性評分 → 覆蓋率正規化加權 `scoring.combine()` → walk-forward 回測網格找權重，採**命中率第一優先**（`pick_balanced` 的 `balance_tol=0`）。
- 所有新面沿用既有模式：`timeline_db` 加表 + `*_asof(date<D)` 無 look-ahead 查詢 → `scoring.DIMENSIONS` 加維度 + `score_*` → `backtest.extract_features/score_samples` 接入 → `apply_guard` 顯著性護欄自動把不顯著面權重歸 0。新面會自動泛化進權重網格、報告、校準。
- **診斷（2026-06，238 交易日）**：現有籌碼訊號方向命中率在**各 horizon 都只有 ~55–58%**：

  | 訊號＼未來 | H1 | H3 | H5 | H10 |
  |---|---|---|---|---|
  | 外資 | 53.2 | 54.5 | 54.6 | 53.9 |
  | 投信 | 55.3 | 55.3 | 52.6 | 56.3 |
  | 三大法人 | 55.7 | 55.0 | 57.3 | 57.1 |
  | 分點合計 | 58.6 | 55.5 | 58.7 | 53.0 |

- 已誠實證實：日內 1 分 K、主力分點（含 walk-forward 行為模型）皆為**真實但弱、且與美光/費半重疊**的訊號，OOS 下無法推升 59.66% 天花板 → 採用權重 0。
- **天花板認知**：單股每日全表態現實上限 ~60%。籌碼要更強，靠的是「結構性持股資料 + 橫斷面廣度 + 籌碼當信心/部位（非隔日方向）」。以下四案依此設計。

---

## 選項 A：TDCC 千張大戶持股比率（第九面）★最優先
**假設**：集保戶股權分散表的「千張大戶（≥1000 張）持股比率」週變化，是大戶默默吃貨/出貨的**結構性**訊號，領先大波段；為單股籌碼裡最可能有真 edge 且我**尚未驗證**者。

- **資料源**：TDCC 集保戶股權分散表（免費、可回補歷史）。
  - 集保 OpenData / 「股權分散表查詢」`smart.tdcc.com.tw`（每週六更新，欄位：資料日期、持股分級 1–15、人數、股數、占比）。
  - 千張大戶比率 = 持股分級「1,000,001 股以上」(或合併 >1000 張級距) 的股數占比；可另算 400–1000 張中實戶。
- **特徵**：`big_holder_pct`（本週值）、`big_holder_chg_1w/4w`（週變化、月變化，吃貨=升）、可選散戶比率背離。
- **look-ahead**：週資料且公布有 lag（約 T+5）。`*_asof` 取「公布日 < D」最新一筆；務必用**公布日**而非資料日比較（資料日為週五，公布在隔週）。
- **實作清單**（仿 `us_market`/`broker_branches` 模式）：
  - `src/fetch_tdcc.py`：抓 TDCC 分散表 → `{date, big_pct, mid_pct, retail_pct}`。
  - `timeline_db`：表 `tdcc_holders(symbol, date, big_pct, mid_pct, retail_pct)` + `tdcc_asof(date<D)` + `counts()`。
  - `scoring`：`DIMENSIONS` 加 `"holders"`；`holders_signals`（big_chg_1w/4w 正規化）+ `score_holders`；參數 `holder_chg_div` 等入 `DEFAULT_PARAMS` 與 `calibrate.CANDIDATES`。
  - `backtest.extract_features/score_samples`：接 `tdcc_asof`；`_DIM_ZH` 加「大戶」；報告/護欄自動含。
  - `ingest`：`backfill_tdcc(start,end)`（逐週回補）+ 每日 `ingest_dataset` 累積；`build_dataset` 加 `holders` 區塊。
- **評估**：回測看大戶面單獨命中率與是否推升天花板；OOS 必過。**預期**：樂觀 +1～2pp；風險：週頻、lag 大、單股仍可能弱。

## 選項 B：籌碼集中度當「信心/部位」濾鏡（不新增資料）
**假設**：籌碼集中度高 + 趨勢一致時，方向更可信 → 用來決定「高信心日才表態/加重」，而非預測隔日。對應之前量化過的「選擇性表態」（|composite|≥0.4 時 ~66%）。

- **資料**：現有 `broker_branches`（集中度 = 前 N 大淨額 / 總成交量；或 Σnet/Σ|net|）與三大法人。
- **實作**：
  - 在 `scoring` 或 `backtest` 增「信心分數」conf(D) = f(籌碼集中度, 訊號一致性, |composite|)。
  - 出兩段式輸出：高信心日給方向（目標 ~65%）、低信心日標「觀望」。報告/技能呈現分級命中率。
  - 注意：**與「每天表態」設定衝突**——需先確認是否接受「部分日觀望」或改為「全表態但標信心等級」。
- **評估**：比較高信心子集命中率 vs 全體；確認非靠降涵蓋率作弊（誠實揭露涵蓋率）。**預期**：高信心子集可達 65%+，但涵蓋率下降。

## 選項 C：外資台指期未平倉（市場級 regime，第九/十面）
**假設**：三大法人台指期淨未平倉（尤其外資）是**市場方向**訊號，影響整個盤含 2344。

- **資料源**：TAIFEX「三大法人－區分各期貨契約」每日報表（免費、可回補）`https://www.taifex.com.tw/cht/3/futContractsDate`。
- **特徵**：外資台指期淨未平倉口數（與其變化）正規化；可加未平倉 vs 現貨背離。
- **look-ahead**：每日盤後公布，`*_asof(date<D)` 取 D-1。
- **實作**：`src/fetch_taifex.py` + `timeline_db` 表 `futures_oi` + `oi_asof` + `scoring` 加 `"futures"` 面 + 接回測（仿美光/費半 `score_us` 的線性轉換最簡）。
- **評估**：與美光/費半可能相關（都是市場 beta）；看是否帶獨立增益。**預期**：市場級訊號，單股增益有限但可能穩定。

## 選項 D：橫斷面多股排序（最大威力，最大改動）
**假設**：籌碼 alpha 主要靠**跨上百檔排序**累積（做多大戶增持/籌碼集中前段、放空後段），單股先天丟失此威力。

- **架構轉變**：從「單股每日方向」→「多股每日按籌碼指標排名」。需要：
  - 多股資料源（candles/chips/TDCC/分點 對 N 檔股票，TWSE/TDCC 皆可全市場抓）。
  - `timeline_db` 改為多 symbol（schema 已含 symbol 欄，但回補與評分迴圈要全市場化）。
  - 新評估框架：cross-sectional IC（資訊係數）、分位數報酬、long-short 回測，而非單股命中率。
  - 大量計算與資料量（數百檔 × 數百日）。
- **產品意義**：這會變成「選股/排序」系統而非「2344 盤前研判」。**工程量最大、與現有產品定位不同**，建議獨立專案評估。
- **預期**：這才是「現代最強籌碼模型」的真正路徑；但需先確認是否要把產品從單股研判轉向多股選股。

## 建議執行順序
1. **A（TDCC 大戶）** — 唯一未驗證、理論最強的單股籌碼訊號，先做、CP 值最高。
2. **C（外資期貨）** — 便宜、可回補、市場 regime，順手加。
3. **B（信心濾鏡）** — 需先和「每天表態」設定取捨；可與選擇性表態一起談。
4. **D（橫斷面）** — 最大潛力但需重新定義產品，獨立評估。

## 共用注意
- 每個新面務必：`*_asof(date<D)` 無 look-ahead（加 assert）；冷啟動覆蓋率低時護欄自動權重 0；回補後跑 `calibrate.bat` 重校；以 `oos_check.py` 為誠實裁決（OOS≥in-sample 才可信）。
- 鐵則不變：命中率第一優先、天花板 ~55–62%、>80% 視為 bug、命中率≠報酬率。
- 參考檔：`src/branch_model.py`（walk-forward+收縮 模板）、`src/fetch_dj_chips.py`（爬取+解析 模板）、`TUNING.md`（調適流程與鐵則）。
